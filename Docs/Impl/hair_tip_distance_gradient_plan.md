# 実装計画書: 髪メッシュ用「毛先から一定距離グラデーション」

## 1. 目的

髪メッシュに対して、**毛先（先端）から一定距離だけ**グラデーションを適用する機能の実装可否を検討し、現実的な範囲での実装計画を定義する。

---

## 2. 実装可否の結論

結論として、**現実的に実装可能**。  
ただし「エッジ角度だけ」で全ケースを安定判定するのは難しいため、以下の方針が必要。

- 毛先判定は単一指標ではなく、**複数ヒューリスティックのスコア合成**で行う
- 距離計算はメッシュ表面上の**グラフ距離（ジオデシック近似）**で行う
- 分岐・合流は「毛先集合からの最短距離」を使うことで自然に扱う
- 判定不安定ケース向けに、**手動補助（選択/除外/閾値調整）**を用意する

---

## 3. 検討項目ごとの整理

### 3.1 毛先の定義（形状差への対応）

#### 課題
- 髪メッシュは太い束、板ポリ、リボン状、房の合流など形状差が大きい
- 「接続エッジ角度のみ」だと、先端以外（折れ曲がり・境界・細部ノイズ）も誤検出しやすい

#### 提案する毛先判定
各頂点に対して次の特徴量を計算し、重み付きスコアで毛先候補を抽出する。

1. **局所開き角（エッジ角度）**
   - 接続エッジ方向の分散・最大角度を使用
2. **境界性**
   - 境界エッジ（片側ポリゴンのみ）に属する頂点を優先
3. **細さ指標**
   - 近傍リング内の断面幅（または近傍距離統計）で先細り傾向を評価
4. **曲率/法線変化**
   - 先端特有の曲率上昇を補助的に利用

最終的にローカル極大＋閾値で毛先集合 `T` を確定する。

> 重要: 判定の自動化率は高くできるが100%固定は難しいため、毛先候補の可視化と手動修正UIを前提とする。

### 3.2 毛先からの距離判定

#### 提案
- メッシュ頂点をノード、エッジ長を重みとするグラフを作る
- 毛先集合 `T` を始点とした**マルチソース最短経路（Dijkstra）**で各頂点距離 `d(v)` を算出
- 距離閾値 `L` に対して `w(v) = saturate(1 - d(v)/L)` を重みとして適用強度を作る

#### 理由
- ユークリッド距離より、毛束に沿った「表面上の距離」に近い
- 分岐形状でも距離場が自然に連続しやすい
- 既存の頂点ベース処理に統合しやすい

### 3.3 分岐・合流の扱い

#### 分岐
- 複数毛先からの最短距離で自然に処理可能
- 分岐点は最短経路の競合点となり、急な不連続を抑えられる

#### 合流
- 合流部で異なる毛先由来距離が混ざるため、必要に応じて次を追加
  - 近傍平滑化（1～2リング）
  - 勝者固定（最短毛先ラベルで離散化）
  - 房ごとの接続成分分離（サブメッシュ/材質単位）

---

## 4. 実装スコープ（現実的な最小構成）

### Phase 1（MVP）
1. 毛先判定（境界性 + エッジ角度）
2. マルチソースDijkstra距離場
3. 距離閾値でグラデーション強度生成
4. デバッグ表示（毛先頂点色、距離ヒートマップ）

### Phase 2（精度向上）
1. 細さ指標・曲率の追加
2. 自動閾値推定（分位点ベース）
3. 分岐/合流部の平滑化オプション

### Phase 3（運用性）
1. 手動補助UI（毛先追加/除外ブラシ）
2. プリセット（ロング/ショート/板ポリ髪）
3. 処理時間最適化（キャッシュ、再計算範囲限定）

---

## 5. 追加が必要な設定項目（案）

- `EnableHairTipDistanceMode`（ON/OFF）
- `HairTipDetectThreshold`（毛先スコア閾値）
- `HairTipMaxDistance`（適用距離）
- `HairTipSmoothing`（分岐部平滑）
- `HairTipManualAdjust`（手動補正モード）

---

## 6. 想定リスクと回避策

1. **誤検出（先端以外が毛先になる）**
   - 複合スコア化、可視化、手動補正を提供
2. **重い処理（高密度メッシュ）**
   - 頂点隣接キャッシュ、設定変更時のみ再計算
3. **メッシュ構造依存**
   - サブメッシュ単位処理、接続成分ごとの独立処理

---

## 7. 判定

本機能は、  
- 自動判定のみで完全解決を目指すのではなく、  
- 自動推定 + 手動補助を前提にすることで、  
実制作で使える品質に到達可能。

したがって、**段階実装（Phase 1→2→3）で進める方針は実現可能かつ妥当**。
