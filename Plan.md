1. システム概要

本ツールは、選択したメッシュのUV座標をキャンバスとし、オブジェクト座標（Object Space）の空間情報を色情報に変換してテクスチャへ書き込む（ベイクする）エディタ拡張です。
2. 機能要件
① グラデーション生成ロジック（コア機能）

    ベクトル指定: グラデーションの方向を Vector3 で指定可能にする（例：(0, 1, 0) なら垂直方向）。

    空間計算: オブジェクト座標における頂点（またはピクセル）の位置座標 P と、指定ベクトル V の内積を用いてグラデーションの強さを算出する。

    正規化（Normalize）: オブジェクトのバウンディングボックス（境界）を基準に、グラデーションの開始点（0.0）と終了点（1.0）を自動または手動でフィッティングさせる機能。

② マスク制御機能

    テクスチャマスク: 既存のテクスチャ（アルファチャンネルやグレースケール）を読み込み、グラデーションの適用強度を制御する。

    頂点カラーマスク（推奨）: VRChatアバターでよく使われる「頂点カラー」をマスクとして利用できる機能。

    反転機能: マスクの白黒を反転させるスイッチ。

③ グラデーションテクスチャの適用

    Gradient Editorの利用: Unity標準の UnityEngine.Gradient フィールドをGUIに実装。

    LUT（Look Up Table）変換: 算出した 0.0〜1.0 の値を、ユーザーが設定したグラデーションの色に置き換えてテクスチャに書き込む。

3. 技術的要件（実装のヒント）
書き込みの仕組み

通常の Texture2D.SetPixel（CPU処理）は非常に低速なため、**GPU（Shader + CommandBuffer）**を使ったベイク手法を推奨します。

    UV描画シェーダー: 頂点シェーダーで Position を UV に置き換えて描画することで、テクスチャ空間に直接書き込みを行います。

    RenderTexture: 計算結果を一度 RenderTexture に描画し、それを Texture2D として保存します。

ユーザーインターフェース (UI)

    Target Renderer: 対象となる SkinnedMeshRenderer または MeshFilter の指定。

    Vector Handle: Sceneビュー上でグラデーションの向きを視覚的に操作できるハンドル（Handles.ArrowHandle 等）。

    Save Path: 書き出し先のパス指定（元のテクスチャを上書きせず、新規保存できる安全策）。

4. ワークフロー（ユーザー体験）

    Unity上でアバターを選択し、本ツールを開く。

    グラデーションをかけたい向き（ベクトル）をSceneビューの矢印で調整。

    UnityEngine.Gradient で色を設定。

    マスクに使用するテクスチャ（または頂点カラー）を選択。

    「Bake & Save」ボタンで、新しいテクスチャが生成され、マテリアルに自動割り当てされる。

5. 拡張性の検討（将来的に）

    プレビュー機能: 「Bake」を押す前に、現在の設定がどう見えるかをSceneビュー上のアバターにリアルタイム投影する機能（一時的なマテリアル差し替え）。

    複数レイヤー: サブスタのように、複数のグラデーションを「加算」や「乗算」で合成できる機能。
